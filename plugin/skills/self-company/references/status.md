### v0.1.13 RAG Quality Overhaul Completed (Phase 24)

- ✅ **Item 1 — multilingual embedding model + model-stamp + retuned floor (THE fix)** — the English-only `bge-small-en-v1.5` compressed every query (any language) into an undiscriminating 0.45-0.65 cosine band, so `RAG_MIN_SCORE=0.30` filtered nothing, and Chinese queries — the Chairman's default language — scored WORSE than random. Swapped to `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` (still 384-dim; `EMBEDDING_DIM` is now DERIVED from a `RAG_EMBED_MODEL -> dim` table in `rag_embed.py`, one source of truth). The index is now **model-stamped** (`rag_stamp.py`, a JSON sidecar): every query goes through `rag_query.py`, which refuses (treats as index-absent) a stale-or-missing stamp — the single choke point every consumer (the ask-time hook, per-employee recall, shared-memory dispatch read) inherits for free. `rag_index.py` self-heals a stamp mismatch by forcing a full rebuild automatically (Phase 12b self-heal pattern) — a plugin update is safe with no manual step. Measured on the real corpus (55 active L1/L2 memories): EN hit@1 0.750→0.875, ZH hit@1 0.375→0.625, off-topic top-1 scores 0.49-0.65→0.15-0.31. `SELF_COMPANY_INJECT_RAG_MIN_SCORE` retuned 0.30→0.35 (data-driven: off-topic ceiling ~0.31, true-positive floor ~0.42).
- ✅ **Item 2 — regression tests close the "off-topic prompt pollutes context" gap for non-English prompts** — `tests/test_hook_memory_inject.py::TestMultilingualRelevanceGate` (venv-gated) proves an off-topic Traditional-Chinese prompt injects nothing and an on-topic one retrieves the right (English-body) memory, end-to-end through the real hook + a real rebuilt index; a historical-proof test confirms the OLD model would have failed the same off-topic case. This surfaced a SECOND, independent bug in `hook_memory_inject.rank()`: its keyword-path recency fallback ("no prompt tokens -> show the freshest memories") conflated a genuinely EMPTY prompt with a non-empty prompt the ASCII-only tokenizer simply can't parse (any pure-CJK prompt) — silently injecting the freshest memories regardless of topic whenever the semantic path degraded. Fixed: the fallback now fires only on a truly blank prompt; a non-Latin-script prompt with no RAG available degrades to nothing, never a false "recency == relevant."

- ✅ **Item 4 — hybrid retrieval (vector + BM25/FTS via RRF), greenlit in-phase** — rode Item 1's one forced rebuild (one schema migration, not two). `rag_index.py` now stores the embedded body text and builds a native LanceDB FTS index on it (a deliberate reversal of "index stores no body" — acceptable because the index is gitignored/local/rebuildable; see `references/rag.md §7`). `rag_query.py` defaults `query_type="hybrid"` (vector+FTS fused via `RRFReranker`, zero new dependencies), degrading to pure-vector automatically on any error (e.g. an older index). **Gate placement (the one real risk, per Elon):** RRF's fused score is rank-fusion, not cosine, and is NEVER returned as `score` — every hit's `score` is always the TRUE cosine similarity (restored from the vector leg, or computed locally against the row's own stored vector for an FTS-only hit), so the SAME relevance floor every consumer already gates on continues to work byte-identically; the off-topic-injects-nothing tests stay green post-fusion. **Honest caveat (Gibby-checked):** on our current ~55-memory corpus hybrid returns byte-identical rankings to pure vector on every exact-identifier probe — the multilingual subword tokenizer already places code tokens well.
